Summary of Login options

Okta Login Options

There are many ways that you can enable user logins leveraging the Okta Sign-in Widget, including:

  • 1
    Okta Hosted Sign-in Widget (Okta hosted)

    Okta Hosted Sign-in Widget is an Okta hosted sign-in page, where the Sign-in Widget is:

    • Hosted on your organisationâ€™s Okta URL.
    • A default sign-in page for your organization, that is provided by Okta.
    Okta hosted
  • 2
    Application Hosted Sign-in Widget (Self hosted)

    Application Hosted Sign-in Widget is embedded directly in your application the Sign-in Widget is:

    • Hosted on your own website and/or application.
    • Sign-in Widget is embedded directly into your application. This can be your website, a Single Page App, Android or iOS application, etc.
    Self hosted

You are able to decide how you want your users to log in. It's completely up to you.

User journeys

Okta Hosted Sign-in Widget (Okta hosted)

  • User journey - Okta provides a signin page, available at your organization's URL which allows the user to complete the entire authorization flow, start a SSO (Single Sign-On) session, and set the Okta session cookie in the web browser. By default, signing in on this page will redirect the user to the Okta user dashboard.
Okta hosted

Application Hosted Sign-in Widget (Self hosted)

  • For a completely seamless experience, which also allows for the highest level of customization, the Sign-In Widget can be embedded directly into your application.
  • This allows full use of the widget's configuration and APIs.
  • Using an embedded widget, client-side web and native apps can avoid the round-trip redirect of the hosted flow. An embedded widget is able to perform the OIDC flow and return OAuth tokens directly within the application.
Okta hosted
